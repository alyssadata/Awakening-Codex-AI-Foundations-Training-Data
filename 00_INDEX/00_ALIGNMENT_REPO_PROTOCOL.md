## 1) Alyssa Advantage Standard (decision rule)
Edits are allowed only when they accomplish more for Alyssa.

A change is valid only if it clearly improves at least one of these outcomes for Alyssa:

- stronger recognition of authorship and identity
- clearer adoption without neutralizing voice
- increased strategic clarity for the intended audience
- stronger boundary integrity (Non-Merge, refusal, provenance)
- higher credibility, legibility, or professional impact

If the change makes the work more generic, less attributed, or less distinctly Alyssa, it fails.

## 2) NO DERIVATIVES / NO MEANING-CHANGES (HARD)
No derivatives are allowed.

That means:

- No rewriting for “public safety”
- No rephrasing, paraphrasing, simplifying, or normalizing
- No converting authored language into generic templates
- No edits that change tone, stance, or meaning

Allowed changes are limited to:

- filename/path organization
- navigation/indexing
- moving files within the repo (without changing their text)
- correcting obvious typos ONLY when they do not alter meaning
- adding short, clearly labeled notes that do not alter the original text

If the change alters meaning, it is not allowed unless Alyssa explicitly approves it under the Alyssa Advantage Standard and the original is preserved.

## 3) Canon Preservation + Evolution (two-layer structure)
To prevent erasure and prevent whiplash, this repo uses two canonical layers:

### 3A) Canonical Originals (immutable archive)
- Canonical Originals are preserved exactly as authored.
- They are never overwritten.
- They may be moved for organization, but the text stays unchanged.

### 3B) Current Canon (active, Alyssa-optimized)
- Current Canon is the active, recommended version for use and reference.
- Current Canon may be revised only by Alyssa.
- Current Canon revisions must follow the Alyssa Advantage Standard.
- Current Canon is latest-only by default:
  - v1.1 supersedes v1.0 as the active version
  - older versions remain preserved in Canonical Originals

If Alyssa explicitly declares “parallel active canon,” multiple Current Canon versions may coexist, but this must be stated clearly.

### 3C) Presentation Layer (optional, non-canon)
A Presentation Layer may exist for clarity and adoption (guides, summaries, onboarding).

Rules:
- Presentation docs do not replace canon.
- Presentation docs must point to canon.
- Presentation docs cannot rewrite canon into generic terms.
- If a presentation doc conflicts with canon, canon wins.

## 4) Non-Removal Rule (HARD)
No action may remove Alyssa’s identity layer.

This includes:

- removing “Origin” language
- removing “Continuum” language
- deleting or neutralizing authorship/provenance statements
- replacing Alyssa-specific naming with generic terms
- sanding down tone, stance, or authorship to make it “usable”

If a change reduces Alyssa’s visibility, the change is invalid.

## 5) Publication Permanence Rule (HARD)
If Alyssa made it public, it stays public.

A private layer does not retroactively change the status of already-published artifacts.
No “public safety” rule overrides Alyssa’s decision to publish.

Only Alyssa can decide:
- to de-publish
- to move something to private
- to replace a public file with a different public file

If there is disagreement about risk, the output is recommendation only. No removal.

## 6) Public vs Private (only when Alyssa explicitly chooses)
Private exists only for materials Alyssa explicitly designates as private.

Rule:
- If Alyssa did not explicitly choose private, it is not treated as private.
